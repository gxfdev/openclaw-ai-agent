# 自我进化机制详解

## 进化原理

小龙虾的自我进化遵循"交互-反馈-学习-优化"的循环。这不是随机变异，而是基于实际交互效果的有向优化。

## 进化引擎架构

```
┌─────────────────────────────────────────────┐
│              进化引擎 (EvolutionEngine)        │
│                                               │
│  ┌───────────┐  ┌───────────┐  ┌──────────┐ │
│  │ 交互日志   │  │ 行为模式库 │  │ 改进建议  │ │
│  │ (perf_log) │  │ (patterns) │  │ (suggest) │ │
│  └─────┬─────┘  └─────┬─────┘  └────┬─────┘ │
│        │              │              │        │
│        ▼              ▼              ▼        │
│  ┌─────────────────────────────────────────┐ │
│  │           进化决策核心                     │ │
│  │  • 分析成功/失败模式                       │ │
│  │  • 提取上下文特征                          │ │
│  │  • 匹配历史模式                            │ │
│  │  • 融合优化回应                            │ │
│  └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

## 特征提取

从用户输入和上下文中提取关键特征，用于行为模式分类：

### 文本特征

| 特征 | 匹配规则 | 模式标签 |
|------|----------|----------|
| 疑问 | 什么/为什么/how/what/why/吗/? | question |
| 求助 | 帮助/帮/help/assist | request_help |
| 情感 | 感觉/feel/emotion/happy/sad/angry | emotion_talk |
| 感谢 | 谢谢/thank/appreciate | gratitude |

### 上下文特征

| 上下文字段 | 示例 | 模式标签 |
|------------|------|----------|
| user_mood | "anxious" | mood_anxious |
| conversation_stage | "early" | stage_early |

### 特征组合

多个特征用 `-` 连接形成模式键，例如：`question-mood_anxious`

## 行为模式学习

### 模式存储结构

```
行为模式 = {
    模式键: "question-mood_anxious",
    回应列表: ["回应1", "回应2", ...],
    上下文列表: [{...}, {...}, ...],
    成功次数: 5,
    失败次数: 1
}
```

### 学习条件

- 交互效果 ≥ 0.7 视为成功
- 交互效果 < 0.4 视为失败
- 只从最近 5 次成功交互中学习
- 需要至少 2 次成功案例才能形成模式

## 回应优化

### 适配流程

1. 提取当前交互的特征 → 生成模式键
2. 查找行为模式库中匹配的模式
3. 如果模式成功次数 > 失败次数：
   - 随机选择一个成功回应作为灵感
   - 将灵感回应与原始回应融合
4. 否则使用原始回应

### 回应融合策略

融合不是简单拼接，而是取两个回应的互补部分：

```
回应1: [前半部分 A] + [后半部分 B]
回应2: [前半部分 C] + [后半部分 D]
融合:  [前半部分 A] + [后半部分 D]
```

保留原始回应的核心逻辑，融入成功案例的补充元素。

## 效果评估

### 有反馈时

直接使用用户提供的反馈分数（0.0~1.0）。

### 无反馈时（启发式评估）

| 指标 | 加分 | 条件 |
|------|------|------|
| 回应长度适中 | +0.1 | 10 < 长度 < 200 |
| 包含积极词汇 | +0.15 | 理解/帮助/建议/方案/idea/solution |
| 基础分 | 0.5 | 默认 |

## 改进建议生成

当存在失败交互时，自动生成改进建议：

```
"对于输入'{失败输入前50字}...'的响应可能不够有效，考虑改变策略"
```

建议数量上限 10 条，超出时淘汰最旧的。

## 进化状态报告

进化引擎可报告以下状态信息：

- 行为模式总数
- 累计交互次数
- 改进建议列表
- 各模式的成功/失败比
